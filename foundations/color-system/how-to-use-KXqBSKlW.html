How to use the system

In the Atlas color system, the scheme is a curated set of color roles that work together to create a cohesive and accessible visual style. Color roles are a color guide that tells you which colors to use for different parts of your app, like the background, buttons, and text. This keeps the design and UX consistent and accessible for everyone.

 

 

 

General concepts for color roles

These are high-level definitions of the Atlas system scheme:

 

Surface

Roles used for backgrounds and other large areas of the screen that don't need to stand out. Think of it like the foundation of your design.

 

Top Level (Primary, Guidance, Intelligence, Commerce, Error)

Roles used to emphasize or de-emphasize foreground elements.

 

On

Roles with this term indicate a color for text or icons on top of its paired parent color.

 

Container

Roles used as a fill color for foreground elements like buttons. They should not be used for text or icons.

 

Fixed

Roles that maintain the same tone in light and dark themes.

 

Dim

Roles that provide a stronger, more emphasized tone relative to the equivalent fixed color.

 

Variant

Roles ending with this term offer a lower emphasis alternative to its non-variant pair.

 

 

 

Color pairings

To make sure our products are easy for everyone to see, it's important to use the Atlas color system color combinations in the right way. Mixing and matching colors incorrectly can cause problems, especially for people who customize their contrast settings. Each color role has a defined set of color pairings, meaning they are assigned to specific elements based on how those elements are used.

 

Do

Pair layer color roles as intended to ensure expected visual results and accessibility. Notice how these combinations stay legible as the contrast level changes:

  1. Primary
  2. On Primary
  3. Primary Container
  4. On Primary Container
Don't

Improper color mappings can produce unintended visual results and break accessibility. Notice how these combinations become illegible as the contrast level changes:

  1. Primary
  2. Guidance
  3. Guidance Container
  4. On intelligence

 

 

 

Primary roles

The primary is used to create clear distinctions in the interface. It helps define visual hierarchy, indicate interactive elements, highlight different states, and emphasize important information.

 

Primary

For important elements that need to stand out against the background, like buttons or key pieces of text

 

On Primary

For text or icons that go on top of a "Primary" color, use this color to make sure they're easy to read

 

 

Primary Container

For the background of really important components, like a floating action button (FAB), to make them prominent

 

On Primary Container

For any text or icons that go on top of a "Primary Container" color

 

 

Primary Fixed Dim

For when a stronger contrast is needed while remaining consistent behavior across themes

 

Primary Fixed Variant

For a lower emphasis alternative to its non-variant pair

 

 

 

Guidance roles

Guidance builds on our brand equity and provides clear wayfinding, enabling users to receive feedback and make informed decisions with confidence.

 

Guidance

For guidance related elements that need to stand out against the background, like buttons or key pieces of text

 

On Guidance

For guidance related text or icons that go on top of a "Guidance" color

 

 

Guidance Container

For the background of guidance related components, like a floating action button (FAB), to make them prominent

 

On Guidance Container

For any guidance related text or icons that go on top of a "Guidance Container" color

 

 

Guidance Fixed

For when a guidance related background color is needed that doesn't change, regardless of whether the user prefers light or dark mode

 

Guidance Fixed Dim

For when a stronger contrast is needed while remaining consistent behavior across themes

 

Guidance Fixed Variant

For a lower emphasis alternative to its non-variant pair

 

 

 

Error roles

The Error color highlights key elements and actions related to Error within the traveler’s experience. This color is also used to draw attention to the best options, provide warnings, give instructions, and guide users through Error related interactions.

 

Error

For Error related elements that need to stand out against the background, like buttons or key pieces of text

 

On Error

For Error related text or icons that go on top of a "Error" color

 

 

Error Container

For the background of Error related components, like a floating action button (FAB), to make them prominent

 

On Error Container

For any Error related text or icons that go on top of a "Error Container" color

 

 

Error Fixed

For when a Error related background color is needed that doesn't change, regardless of whether the user prefers light or dark mode

 

Error Fixed Dim

For when a stronger contrast is needed while remaining consistent behavior across themes

 

Error Fixed Variant

For a lower emphasis alternative to its non-variant pair

 

 

 

Surface roles

Surface colors are ideal for neutral backgrounds, while its "container" colors work well for elements that need to stand out a bit more, like cards, sheets, and dialogs. This helps create visual layers in the product design. Note: There are no fixed roles for surface colors.

 

Surface

Default color for page and large component backgrounds

 

On Surface

Text and icons against any surface color

 

On Surface Variant

Lower-emphasis color for text and icons against any surface color

 

 

Surface Container Lowest

Lowest-emphasis container color

 

Surface Container Low

Low-emphasis container color

 

Surface Container

Default container color

 

Surface Container High

High-emphasis container color

 

Surface Container Highest

Highest-emphasis container color

 

 

 

Outline roles

There are two outline colors to be used against surface.

 

Outline

For important boundaries, such as those around text fields, to define and separate them from surrounding elements

 

Outline Variant

For decorative elements that visually separate different sections of content, like subtle lines or borders

 

 

 

Inverse roles

Colors that let you create contrast by flipping the usual color scheme.

 

Inverse Surface

Use this for backgrounds that need to stand out from the normal background.

 

Inverse On Surface

This is for text and icons that go on top of an "inverse surface," making sure they're still easy to see.

 

Inverse Primary

This is used for important actions, like buttons, when they're placed on an "inverse surface." This helps them pop and get noticed.
