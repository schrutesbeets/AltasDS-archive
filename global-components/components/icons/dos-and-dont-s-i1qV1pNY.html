Icons

Icons serve as a visual language, providing a consistent and recognizable set of symbols to communicate actions, concepts, or status across an interface.

Design guidelines

Consistent use of stroke widths, style and metaphor is necessary for icons to feel streamlined in the Atlas Design System.

Do

Draw simple objects that are easy to identify.

Don't

Don't use embellish icons with unnecessary details.

Do

Create icons that are outlined.

Don't

Don't use fill the icons, unless that icon has an interactive state.

Do

Ensure that icons are two-dimensional, and objects face forward.

Don't

Don't use perspective and 3D objects.

Design Tip: When creating an icon, view and edit it at 400% (192 x 192 dp), which will display edges at 4dp. By maintaining this ratio, any changes to the original will be scaled up or down proportionally, which preserves sharp edges and correct alignment when the scale is returned to 100% (48dp).
Do

Reuse parts of other icons in the set to maintain visual harmony across all icons.

Do

Use simple geometric shapes, angles, and rounded corners.

Don't

Don't use excessive details or organic shapes.

Do

Use filled icons for semi-permanent selected states, like giving a thumbs up.

Don't

Don't use filled icons to create emphasis.

Do

Use a consistent visual style, stroke weight and only one color.

Don't

Don't use transparency.

 

 

 

Grid and keylines

The icon grid and keylines ensure consistent sizing of icons in the Atlas Design System.

All (redesigned) icons are designed on Google Material icon grid using keyline shapes. The icon grid establishes clear rules for the consistent, but flexible, positioning of graphic elements. The grid enables us to size icons consistently. All icons are designed at 24x24px and can be scaled to any of the approved icon sizes noted in the sizing section above.

Keyline shapes are based on the grid. By using these core shapes as a baseline, you can maintain consistent visual proportions throughout your product icons:

The icon container is 24 Ã— 24px.

The content of an icon should be confined within a safe space of 20 Ã— 20px.

Stroke weight is 1.5px. Shapes must align with the pixel grid.

The minimum gap between strokes should never be less than 1px.

Corner radius can go from 1px to 3px, depending on the desired roundness of the object. Joints and terminals must be rounded. Sharp corners are not allowed, except for intersections and cutouts.

Strokes are a mix of rounded and squared off, even when a shape cuts out into another.
