Map Pin

A universal, default component with dynamic behaviors that visually represents points of interest (or POIs) on a map, allowing users to interact with and quickly access relevant information.

Also known as Pin, Marker, POI marker

  • Overview
  • Dos & Don'ts

Best practices

 

Map Pin Anchor

Do

Use the Map Pin Anchor when a POI needs to be distinct and recognizable within a map that is visually dense. It should appear as the top layer in the pin layering hierarchy.

Don't

Don't use the Map Pin Anchor to indicate more than one POI. A Map Pin Anchor should serve as a clear focal point among other POIs.

 

Usage

Do

Only use Map Pin components in the context of a map use case.

Don't

Donâ€™t place the Map Pin component outside of a map view. Without a map, the component loses its meaning and can look like a confusing, non-functional element.

 

Size

Do

Map Pin components meet minimum touch target sizes and collision logic has been implemented within the map experience.

Don't

Don't change the scale or change the default size or hover state of the map pins. They have been specifically designed and optimized for all screens sizes and devices.

 

Color

Do

The default colors of the Map Pin component are specifically chosen to maintain a high contrast ratio on a map surface in both light and dark modes.

Don't

Don't change or override the default outline, fill, or text/icon colors of the Map Pin component. Altering these will negatively affect accessibility, especially in both light and dark modes.

 

Icon

Do

The Map Pin Maximized is intended to be used with specific iconography of POI types that are supported by our map platform.

Don't

Don't use obscure icons that are not representative of the POI they are referring to. Stick with the default icons as much as possible.

Do

The icon in the Map Pin Anchor should represent an icon that no other Map Pin component has in order to remain distinct.

Don't

Don't change or override the default icon of the Map Pin Anchor.
