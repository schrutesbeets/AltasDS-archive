Elevation

Atlas DS follows the Google Material default values for elevation offsets (listed in the table below). The bolded components are components found in web or in the native Tripadvisor apps, as well as unique Tripadvisor components that are not in Google Material. Though all components have an elevation value, for Tripadvisor they do not all have an associated shadow. If the component has a shadow, it is noted in right hand column.

Semantic elevations

A semantic name describes the functional use of the elevation, rather than the elevation itself. For example “datePickerElevation” (semantic name) and “elevation16” (elevation design token) both have an elevation of 16 but “datePickerElevation” specifically describes the elevation of the date picker.

General guidelines

Semantic elevations are prescriptive, each one has an intended use. This allows more targeted customizations. For example, you could change the elevation of the FAB without affecting the elevation of the Map Card, even though they share the same elevation token because they use different semantic elevations.

Semantic elevations also enable us to easily adjust the design system for different themes.

(Work in progress): Each Design System component is built using only semantic elevations, we highly recommend using semantic colors wherever possible.

 

 

Naming convention

<component> + “Elevation”. The semantic name is written in camel case. Do not include numbers, special characters or punctuation.
