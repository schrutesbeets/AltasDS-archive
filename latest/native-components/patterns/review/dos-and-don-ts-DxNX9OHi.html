Review

A review is a piece of user content that evaluates a property or product.

  • Overview
  • Dos & Don'ts

Properties

 

Header

The header property defines the layout of review headers with two variants. Contributor reviews pair a three-line contributor primitive with circular action buttons, with menu options that adapt by role (owner, traveler, or review writer). User reviews, shown in the reviews tab, use a simplified header optimized for displaying a user’s own contributions.

 

Review content

The review content is the core of the review pattern. It includes a required title, rating, date shared, and the main review text (presented with the collapsible text component). An optional label may also appear if the review is marked as a hotel or restaurant favorite.

 

Images

If a user uploads images to their reviews, they are displayed below the main text content.

 

Review metadata

The review metadata surfaces supplementary details captured during the review process. It can include information such as the date visited, type of visit, and other contextual details that provide additional insight into the reviewer’s experience.

 

Footer

The footer includes the like/helpful action and is required for all reviews.

 

Management response

Management responses are replies from a property or product owner or manager. They appear indented beneath the corresponding review and include a contributor primitive, body text, and the date written.

 

 

 

Usage

 

Traveler reviews

Best practices

 

Order

Do

Keep the predefined order of information for reviews.

Don't

Do not change the order of information.
