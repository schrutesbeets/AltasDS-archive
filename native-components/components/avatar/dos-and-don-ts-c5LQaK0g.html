Avatar

An avatar is a visual representation of a traveler or business in a user interface.

  • Overview
  • Dos & Don'ts
  • Implementation

Properties

 

Badge

A badge can be applied to only x-large default and image avatars. They are utilized when a user wants to edit their avatar photo.

 

 

 

Usage

 

Default

While we automatically assign a default image to users, in the case that an image is not assigned, new users are given an illustrated avatar. It has no personalized attributes.

 

Image

We automatically assign a default image when a new user signs up for an account and does not upload their own photo.

Users can also personalize their avatars to better represent themselves.

 

Overflow

The overflow variant appears only after 3 default/image are present. The number is variable to the total number of users within its context, ie: users added to a Trip. This component is only supported for medium and large sizes.

 

 

 

Behavior

 

Interaction

In most cases, an avatar alone is not interactable. It must be within a contributor to be tappable.

However, when paired with a badge, the avatar may be clickable. The tap target for the badge includes the avatar as well.

 

 

 

Best practices

 

Sizing

Do

Use the provided avatar sizes.

Don't

Do not create custom sizes.

 

Overflow text

Do

Only enter numbers within the overflow variant.

Don't

Do not include letters or special characters within the overflow variant.
