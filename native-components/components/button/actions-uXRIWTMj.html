Actions

Interactive elements that allow users to perform tasks or trigger events. They are typically represented by buttons, links, or other interactive elements that respond when a user taps.

 

 

 

Guidelines

 

Actionable

A button label should clearly describe the action that occurs when tapped, even if it’s simply acknowledging something.

Contextual

Buttons interact with other elements on a screen to highlight the most important actions the user needs to take in that context.

Concise

Buttons capture the user’s intent, encapsulating the action a user wants to take in a clear and concise manner.

 

 

 

Types

There are three types of buttons: Button, Icon button, and FAB.

 

 

 

Which button should I use?

 

Button

Buttons are controls that let users take action, make choices, and move forward.

 

Icon button

Icon buttons help users take supplementary actions in a compact area.

 

FAB

FABs, or floating action buttons, are used on screens with long scrolling views that require persistent access to an action or anchoring to a specific location on the page.
