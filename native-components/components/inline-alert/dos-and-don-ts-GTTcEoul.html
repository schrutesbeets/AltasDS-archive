Inline Alert

An inline alert is a stylized, interactive piece of UI. It displays a prominent message and provides users with information.

  • Overview
  • Dos & Don'ts

Properties

 

Label

The alert label is used to highlight the main message of the alert.

 

Alert description

The alert description should describe what the alert is about. Include an text link to provide users with more context on the nature of the alert.

 

Sponsor

Sponsors indicate that a third party has sponsored the message. It can only be included on primary alert variants.

 

 

 

Behavior

 

Alert stacking

A maximum of two alerts can be stacked on top of each other.

 

Text wrapping

While alerts always display the full text, they should concise. We recommend limiting the character count/number of lines as follows:

 

Label

Description

1-3 words

150 character / 3 lines

 

 

 

Best practices

 

Content

Do

Inline alerts must convey an update, warning, success or danger message from our Trust & Safety team regarding travel to a specific destination or POI.

Don't

Do not use an inline alert to convey any other identifying, marketing or propositional information related to a destination or POI.

 

Learn more

Do

Include an text link to provide users with more context on the nature of the alert.

Don't

Don't use an inline alert without a text link. Don't use verbose language to explain the context of the alert within the message.

 

Sponsor

Do

Use the Sponsored variant to indicate a third party has sponsored the message.

Don't

Do not add sponsorship to the Warning, Success, or Danger variants.
