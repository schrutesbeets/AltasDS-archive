Dropdown

Dropdown fields allow users to choose a value from a predefined list.

  • Overview
  • Dos & Don'ts

Properties

 

Label

Labels are required for dropdown fields and describes the field’s purpose.

 

Placeholder text

Placeholder text remains visible until a character is entered or a value is selected. It should provide helpful guidance on the type of input that should be entered.

 

Preselected text

In some cases, dropdown fields may display a preselected value to reduce effort and speed up task completion. Preselection is typically used when there is a most common or highly expected choice.

 

Error messages

Helper text displays an error message to guide the user in meeting input requirements. Error messages are brief and straightforward, taking the place of any previously shown helper text.

 

 

 

Usage

 

Single select

Use a dropdown field when a user needs to select one option from a predefined list (7 items or more).

Dropdown fields are best for standardized values such as country, language, or currency.

 

 

 

Behavior

 

Interaction

Interacting with a dropdown field opens a sheet or a fullscreen modal. The type of modal depends on the length and complexity of the options list.

 

Overflow

Content that extends beyond the width of the field is truncated before the dropdown icon.

 

Placement

When stacking related text and dropdown fields, always keep a 16px spacing between them.

 

 

 

Best practices

 

Label

Do

Ensure the dropdown field has a visible label.

Don't

Do not remove the label from the dropdown field.

 

Selection

Do

Use dropdown fields for a single choice from a predefined list.

Don't

Do not use dropdown fields for very few options, radio buttons are usually clearer.
