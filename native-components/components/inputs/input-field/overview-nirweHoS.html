Input Field

Input fields allow users to enter the number of guests, specify the number of rooms, or select a single date or a range of dates.

Guidelines

When a user clicks an input field, a picker, displayed in a bottom sheet, opens to allow a selection. Once a selection is made, the Input Field updates to reflect it. Unlike Text Fields, Input Fields do not allow freeform input; users must make a selection through the Picker.

 

 

 

Types

There are two types of input fields: Multi-input and Segmented. Learn more about when and how to use each.

Anatomy & Layout

 

Single input

A single input field is the building block within a multi-input field. Each one captures a specific type of information, such as a date, number of rooms/guests, or time. It consists of a container with a required leading icon, a label or placeholder, and a value display that updates after selection.

 

Multi-input

A multi-input field is a container that groups multiple single input fields into a cohesive unit, allowing users to input or select related information in one place.

 

Segmented input

A segmented input field is a single container that organizes multiple pieces of related information, such as date, number of rooms/guests, or time. Dividers are required in order to separate each segment, providing visual clarity, while each segment displays a label or placeholder and a value once selected.

 

 

 

States & Appearances

 

States

Input fields use state layers for interaction states. The available states are enabled, pressed, and focused, and selected. Selected state is only available for single input fields as segmented input fields prompt the interaction of a picker.
