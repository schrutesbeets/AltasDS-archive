Text Field

Text fields allow users enter and edit one line of text.

  • Overview
  • Dos & Don'ts

Properties

 

Label

Labels are required for all text fields and describes the field’s purpose.

 

Required/Optional fields

Mark required fields with an asterisk. For fields that aren’t required, show “optional” in parentheses next to the label.


If the majority of fields are required, only label the optional ones. If most are optional, only mark the required fields.

 

Placeholder text

Placeholder text remains visible until the user enters one or more characters. It should provide helpful guidance on the type of input that should be entered.

 

Helper text

Helper text is placed below the field and informs the user of any requirements or disclaimers.

 

Helper text will be replaced with an error message if an error occurs.

 

Character count

Character counter appears when a field has a character limit and updates in real-time as characters are added or removed. Once users reach the maximum, additional input is blocked and triggers an error state and message.

 

Trailing icon

Trailing icons appear at the end of a text field to provide additional context or functionality. These icons help communicate purpose, enhance usability, and give users quick access to relevant actions without taking up extra space.

 

 

 

Usage

 

Default

The default text fields are the most standard field used for general text entry. They can support labels, placeholder text, and helper text to clarify the expected input.

 

Password

The password text fields are a secure input field that masks entered characters to protect sensitive information. It often includes requirements in helper text and provides a "show/hide" eye toggle.

 

Phone number

The phone number text field is formatted for numerical entry, often with built-in validation for length and characters. It includes an optional country code support.

 

Search

The search text field is optimized for entering search queries.

 

 

 

Behavior

 

Inputting format

Input formatting helps clarify expected values and improves readability by automatically adding slashes, dashes, or spaces as the user types.

 

Overflow

Text field content can overflow horizontally. While typing, the field scrolls to keep the cursor visible. When the field loses focus, the content scrolls back to the start, and refocusing returns the view to the end.

 

Read-only

Users cannot focus on or change read-only fields. Their input is derived from other parts of the experience and is intended to be viewable for reference. The input is submitted with the form.

 

Placement

When stacking related text fields, always keep a 16px spacing between them.

 

 

 

Best practices

 

Helper text

Do

Use helper text for important information as it helps users understand how to complete the text field or to indicate any needed input.

Don't

Do not put helper text within the placeholder field.

 

Label

Do

Ensure the text field has a visible label.

Don't

Do not remove the label from the text field.

 

Error messaging

Do

Provide clear and useful error messages that help the user fix the issue.

Don't

Do not display generic error messages.

 

Required/Optional labeling

Do

Only mark the less common fields. If you have five required and one optional, indicate the optional field to reduce unnecessary cognitive load and vice versa

Don't

Do not mark all required field with an asterisk, and vice versa with optional fields.
