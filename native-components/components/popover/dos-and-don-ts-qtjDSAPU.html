Popover

Popovers are brief, informative messages used to introduce new content and features or add additional context to other UI elements. They appear when a user lands on the page, and will persist until the user dismisses them.

  • Overview
  • Dos & Don'ts

Properties

 

Title

The title describe the purpose of the popover. It can be used to expressively message the callout.

 

Message

Message content is required. It is used to explain the purpose of the popover and orient the user.

 

Dismiss button

The dismiss button closes the popover.

 

Pointer/Caret

The pointer is the visual link between the UI element and popover. The pointer can appear along the right, left, or center of the container.


Native popovers only support positioning above and below the popover container.

 

Placement

The pointer position depends on the location on the parent element. They sit an 8px distance between the point of origin.

 

 

 

Behavior & Display

 

Presentation

Static popovers automatically appear when a user lands on the page. They are used to direct a user’s attention to specific information or to a certain UI element. Once a user dismisses a static tooltip, there is no way to trigger it again within the same session.

 

Dismissal

Popovers will persist until they are dismissed by the user or if a user interacts with the elements on the screen. The minimum tap target for icon button are 48x48px.


If a user does not interact with the popover or screen, the popover will auto-dismiss after 5-7 seconds.

 

Text wrapping

It’s recommended that the title of a tooltip take up no more than 1 line and that the content/message take up no more than 3 lines.


The tooltip does not support truncation as these messages should be kept short and brief and ensure the user can read all available information.

 

 

 

Customizations

The popover’s caret is available in left, center, and right alignments by default, but you can customize its position by modifying the horizontal padding in the auto layout settings of the caret position layer.

 

To adjust the caret position , drill into the component and select the caret position layer.

 

In the auto layout section of the Design panel, adjust the horizontal space until the caret is in your desired position.

 

 

 

Screen sizes

 

Small

On compact screens, popovers can only floating above and below the pointed element. They remained either aligned left, right, or center depending on the pointer alignment.

 

Medium

On medium screens like tablets, popovers anchor above or below to the pointed element.

 

 

 

Best practices

 

No Assets

Do

Popovers should not include any leading icons/imagery.

Don't

Do not include leading icons/imagery within popovers

 

Frequency

Do

Only show one popover at a given time.

Don't

Do not show multiple popovers.

Do

Display the popover before any other messaging.

Don't

Do not show multiple forms of messaging at the same time.

 

Interaction/Scrolling

Do

Popovers are dismissed when a user interacts with other elements on the screen, such as scrolling or clicking on a card.

Don't

Popovers should not remain persistent once a user interacts with other screen elements.
