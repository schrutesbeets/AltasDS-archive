Sheet

Bottom sheets are containers anchored to the bottom of the screen that display supporting content or short supplementary actions on a mobile experience.

  • Overview

 

Guidelines

Bottom sheets should be used only within mobile experiences. They are used to temporarily break away from the main context in order to show supplementary content or a short task.

 

Complementary

Bottom sheets complement the content behind them. They provide supporting content or actions that enhance the primary screen without taking users out of their current flow.

Responsive

Bottom sheets respond to the size of the screen and height of the content that they inhabit, adjusting accordingly to their environment and content.

 

 

Types

There are five types of bottom sheets: Promotional, Informational, Selection, Action, and Quick Input.

 

 

Which sheet should I use?

If you want to use a bottom sheet for your design, the content must fit within one of these five sheet types.

 

Promotional sheet

Promotional bottom sheets highlight marketing content, special offers, or calls to action.

 

Informational sheet

Informational sheets provide supplemental details relevant to the traveler’s current browsing context

 

Selection sheet

Selection sheets present a set of options that allow travelers to make choices, configure preferences, or tailor their experience within a given flow.

 

Action sheet

Action sheets present a small list of mutually exclusive actions in response to a user-initiated trigger.

 

Quick input sheet

Quick input sheets capture simple, brief user responses or actions without interrupting the main workflow.

 

Status: The framework for quick input sheets are in progress. Documentation will be added when the component is ready for release. If you need to use this type of sheet, please contact the Atlas Native DS team.
