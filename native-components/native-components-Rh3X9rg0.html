Native Components

The Atlas native library is a human-centered system that promotes clarity and consistency, empowering teams to create seamless experiences for travelers.

Components

An extensive set of components for use in building larger native experiences and patterns.

Avatar

A visual representation of a traveler or business in a user interface.

Avatar Group

A horizontal cluster of avatars representing multiple travelers.

Button

Allow users to perform tasks or trigger events.

Card

Contained units of information and actions related to a single subject.

Chip

Compact elements that present contextual, supplemental options.

Collapsible Text

Allows text to expand and collapse inline.

Contributor

Displays a member of the community who has shared content like reviews or photos.

Date Picker

Allows users to select a date or range of dates depending on the context in which it appears.

Dialog

Blocks interaction with the rest of the app until the user takes an action or dismisses it.

Histogram

Displays the distribution of ratings and review to help users understand feedback at a glance.

Inline Alert

Displays a prominent message and provides users with information.

Inline Prompt

Provides users with information and can compel them to click it for more information.

Inputs

Allows users to provide data or make a selection within a form or interface.

Label

Compact identifiers that highlight key information to help users quickly understand details at a glance.

Labeled Row

Allow users to easily scan and interact with navigational items.

Media Container

A flexible container that displays a variety of visual content.

Navigation Header

Displays common navigation, a global search bar, page titles, and page-level actions.

Number Label

Appear on cards and itinerary pages to indicate a location’s position in the route.

Pagination Dot

Used to indicate the number of images and the user’s current position within the sequence.

Popover

Brief, informative messages used to introduce new content, features, or add additional context to other UI elements.

Progress Bar

Visual indicators of progression through a process or experience.

Search Field

Allow users to enter a search query using custom characters and keywords.

Section Header

Organize the page into scannable chunks of content by providing clear guideposts.

Sheet

Containers anchored to the bottom of the screen that display supporting content or short supplementary actions.

Spinner

Visual loading indicator that a process is in progress but the interface is not yet ready for interaction.

Tab

Navigational elements that allow users to move easily between different sections of a screen.

Tag

Compact, contextual element used to categorize, identify, or provide metadata for content.

Toast

Provide brief feedback to users after taking an action.

Typeahead Result

Interactive pieces of UI that contain content that is created by search engines in response to a query.

 

Patterns

Reusable patterns that address common user needs across native experiences.

Empty State

Appears when expected content isn’t available or cannot be displayed.

Loading State

Inform users that a process is underway.

Media Gallery

Displays collections of images, videos, or other media in a structured and navigable layout.

Review

Piece of user content that evaluates a property or product.
