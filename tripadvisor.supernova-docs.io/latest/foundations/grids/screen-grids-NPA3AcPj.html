Screen grids

The foundation to every layout in this system is the 8-pixel square mini unit. See Spacing for more details.

 

General guidelines

Every screen in the app or site is built on top of a Screen Grid. Each page has a layout of margins, columns, and gutters.

  • Gutters remain fixed at different screen widths.
  • Columns and Margins can be fluid or fixed at different screen widths. The following is a list of breakpoints and the details that are at each viewport width

Grid variants

Different parts of the site or different teams may support different grids. The ones above are our standard set, but it should be possible for teams to use DS components while still defining their own grids when needed, such in the case of B2B. Use cases for different grids should be discussed with the DS team before implementing.

 

 

 

Data references

Tripadvisor does not publish traffic breakdowns by specific screen widths over time, but analysis from industry sources and global screen resolution statistics provide important context (sources: 1, 2, 3, 4, 5):

  • Tripadvisor’s device share (2024–2025):
    • Mobile devices: Consistently ~62–63% of traffic.
    • Desktop: 37–38%.
    • Tablet: Less than 2%, globally.

 

Screen Widths and Their Share (2025, industry-wide, not Tripadvisor-specific):

  • The most common screen resolutions worldwide as of mid-2025:
    • 1920x1080: 7.57%
    • 360x800: 7.05%
    • 390x844: 3.76%
    • 393x873: 3.68%
    • 1536x864: 3.34%
    • 1366x768: 3.1%
  • Mobile traffic is dominated by screen widths between 360 px and 414 px.
  • Desktop traffic is led by widths such as 1366 px, 1440 px, 1536 px, and 1920 px.

 

Change over time:

  • Prior to 2020, desktop accounted for a larger share, but from at least 2021 onward, mobile became dominant, growing as high as 62–63% by 2025.
  • This rise tracks the global trend of mobile overtaking desktop for internet traffic. For example, in 2016, industry estimates saw less than 25% of Tripadvisor’s web views from mobile, but by 2025, mobile clearly leads.

 

 

 

Breakpoints

Breakpoints are points where the site or app content responds according to the device or container width, allowing you to show the best possible layout to the user.

Breakpoints are typically determined by viewport, and the sizing corresponds to the viewport sizes laid out above, however breakpoints can also be used independent of the viewport for container sizes within viewports.

Design usage guidelines

The grid system also allows for grids to be defined and nested within element containers, such as cards, carousel and grid shelf layouts. This allows the pattern to fit within the page constraints, but also gives it flexibility to allow for dynamic UI design.

1. With a frame selected, click the Styles icon in the Layout Grid menu

 

2. Find the Page Layout Grid that applies to screen you are designing for

 

3. Select the Page Layout Grid to apply to it your selected frame (pro tip: hover over the style name to see what viewports that grid covers)
