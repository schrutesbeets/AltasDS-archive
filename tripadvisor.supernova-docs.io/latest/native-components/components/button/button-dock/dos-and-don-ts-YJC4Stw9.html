Button Dock

Button docks are buttons inside a container that stay on the screen as a user scrolls. They keep buttons visible on a screen at all times, allowing users to quickly decide and act at any moment without having to scroll to the end of page.

  • Overview
  • Dos & Don'ts

Behavior

 

On scroll

When a stacked button dock is placed over a scrollable view, there is a divider to show that there is more content below the fold.

 

 

 

Screen Size

 

Small

On small screens, the button dock maintains its narrow structure.

 

Medium

On medium screens, such as tablets, the button dock adjusts to optimize spacing and hierarchy. If there is a dismissive CTA, it will be left aligned.

 

 

 

Usage

 

Stacked

Use the stacked layout when multiple actions need to be shown in order of priority. It works well for complex flows where secondary or dismissive options support a clear primary action.

 

Side-by-Side

Use the side-by-side layout when presenting a primary action alongside a single alternative, such as a cancel or dismiss option. This format is best for simple, binary choices.

 

Link

Use the external link layout when only a lightweight, secondary action is needed, such as linking out to related information or resources without emphasizing a primary action.

 

 

 

Best practices

 

Number of buttons

Do

Only stack a maximum of four buttons in a stacked button dock.

Don't

Do not stack more than four buttons.

 

Consistent size

Do

Keep the buttons within the button dock their default size.

Don't

Do not change the sizes within the button dock.
