Contributor

A contributor displays a member of the community who has shared content like reviews or photos.

  • Overview
  • Dos & Don'ts

Properties

 

Avatar/Avatar group

An avatar is required for all contributor types. Default and bio contributors support only a single avatar, while attributor contributors can display either a single avatar or an avatar group.

 

Icon

The attribution contributor component supports an optional icon property. Icons can be used to visually indicate the type or source of a contributor, such as Powered by AI.

 

Contributor content

The content properties define the textual and informational elements of a contributor. These properties provide the essential details (such as the contributor’s name/credit) as well as optional supporting metadata.

 

Trailing menu

A trailing menu is an optional element that appears alongside a default contributor. When selected, it reveals a list of actions or additional information related to the contributor.

 

 

 

Usage

 

Attributions

Use the contributor component to credit content sources. Always include the contributor’s name, label, or metadata like contributor count. Keep it concise and clearly associated with the content.

 

Reviews

Display contributors who provide user reviews or ratings with their name or username, and optionally, relevant metadata like review count or location. Contributor info should support the review content without overpowering it, making it easy to scan.

 

Profiles

Use the component to showcase a contributor’s personal or public profile, including name, avatar, and optional bio or expertise. Metadata such as joined date or contribution count can provide context. Ensure the layout is clean and readable across devices.

 

 

 

Behavior

 

Tap target

The default contributor (avatar and content only) is interactive and when tapped, navigates users to the user’s profile.


The trailing menu has a separate tap target related to the icon button.

 

Text wrapping

The secondary/metadata content within contributors truncate after 1 line of text. Some variations allow for stacking to accommodate more information.

Best practices

 

Credit

Do

Always include the contributor name.

Do

Do not hide the contributor name.

 

Interaction

Only small and medium default contributors are interactive.
Do

Do make the component interactive only when it provides clear value, such as linking to a contributor’s profile, review history, or related content.

Don't

Do not make the component interactive unless it clearly leads to a profile or contributor page.
