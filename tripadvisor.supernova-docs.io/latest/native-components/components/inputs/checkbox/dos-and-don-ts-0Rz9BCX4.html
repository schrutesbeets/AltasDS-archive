Checkbox

Checkbox is a control that allows users to select multiple options from a list or a single yes or no option.

  • Overview
  • Dos & Don'ts

Properties

 

Label

A label is required for all checkboxes. They should be short or represent a yes or no decision.

 

Secondary text

Checkboxes can include secondary text (subcopy) beneath the main label to provide additional context. This is helpful when the primary label alone may not fully explain the option.

 

Trailing checkbox

Checkboxes are trailing by default to prioritize reading the content first, making them ideal for filters, preferences, and settings where users interpret labels before acting.

 

Leading checkbox

Leading checkboxes are used for binary yes/no choices where the checkbox itself is the primary action.

 

 

 

Usage

 

Filters

Use checkboxes in filters when users can select multiple options from a list to refine results. Each selection is independent, and checking one option does not affect others.

 

Single-select

A single discrete checkbox can be used for yes/no decisions within a form.

 

 

 

Behavior

 

Label overflow

Labels should be kept short, but will wrap if they are wider than the container. The checkbox remains aligned to the top of the text box.

Best practices

 

Selection

Do

Use checkboxes for multi-selection of related list items.

Don't

Do not use checkboxes when only one option can be chosen at a time.

 

Alignment

Do

Keep checkbox alignment consistent when listing out related items.

Don't

Do not mix leading and trailing checkboxes in one list.

 

Checkbox vs Switch

Do

Use a checkbox for an option saved after a final button is tapped.

Don't

Do not use a switch when there is a final button to submit the selection. Use a switch from immediate effect.
