Buttons & CTAs

Quick tips on CTAs

Overview

CTAs—in-line or button—are always sentence case, and do not get punctuation. This also goes for CTAs at the end of shelves or disclaimers, like Learn more or See terms. 

Anatomy 

Length: Keep short: 1–3 words, under 25 characters. 
Tone: Action-driven, clear. Be direct about what will occur or be shown as a result of clicking the CTA. Use verbs when you can.

Variations

Consistency is great, but variation is not a crime. While having some standardization on things like shelves, checkouts, and sign-ins is crucial, it’s fine to switch up things based on context and space (Next vs. Continue; Back vs. Go back).

Common buttons & CTAs

Display  & expansion

See all / See more 
Typically done for CTAs that link to list pages or group/category pages. See all is the default for shelves that link out to list pages.


Show more / Show all / Show less
Use for displays, content, and modules that expand/contract on the same page. 


Read more / Read [reviews] / Read full [post] 
Use when leading users to more content-heavy areas where full reading might be done, like a review, tip, or forum post. Be specific when possible. 

Purchase & commerce

View cart 
Displays all items inside the user’s cart. Mostly used on the Partners side, as experiences is currently cartless.


Continue to checkout / Check out 
Advances the user through the purchase funnel. Shorter version appropriate for mobile or smaller spaces.


Place order 
Used to complete a transaction in Partner checkout. 


Complete booking (hotels/exp) / Complete reservation (restaurants)
Finalizes reservations/purchase.

Navigation

Back / Go back 
Takes users to previous states/screens. Shorter version more appropriate for input flows (e.g., GAI, Tips). The longer version is better for dialogue modules (as an alternate to Cancel). 


Next 
Advances the user to the next state/screen in a flow.


Continue
Advances the user in a flow; best used when advancing includes acceptance of company/offer terms. 

In-line text links

Learn more
Used primarily on shelves to link to site terms, sort explanations, or program landers. 


See terms / See [Policy/Contract]
Used primarily for T&Cs in legal copy/disclaimers.

A note on in-line CTAs: Keep these to a minimum. While “Learn more” functions well enough for our shelves now—succinct, in context—the inline CTA should ideally be describe what will proceed. “See Tripadvisor’s Privacy Policy” is an example. This practice helps all users, but it’s particularly helpful for screen readers. Learn more about in-line CTAs

BU-Specific CTAs

Restaurants

Reserve / Reserve a table 
Shorter version used for list pages; longer version used for /RR pages.


Order online / Order now
Used for Grubhub-partnered restaurants. 


Find a table / Make a reservation 
Currently used for restaurants that don’t use OpenTable as their reservation partner, and for restaurants with no same-day availability. 

Experiences

Reserve / Reserve now 
Shorter version used for list pages; longer version used for /APR pages.


Check availability
Used for undated features, like the sticky nav on /APR pages. 

Hotels

Check availability / Show prices
Used for undated landing/list pages. Latter used coming from search; prompts date entry.  


View hotel 
Used on list pages for individual properties.


View deal / View rooms / Reserve [#] rooms
Used across the booking funnel.

UGC

Upload a photo / Add a photo
“Upload” is used in most CTAs for photos, but “add” is used for the empty state and on the partner side. 


Write a review / Review
The longer version is most frequently used, but one-word CTA works for smaller spaces, like a nav or alongside an icon.


Share an insider tip / Add an insider tip
Can shift based on location. Can drop “insider” once we’re deeper in the user funnel or if we have insider in the headline.


Ask a question
For posting a question/directing to forums from /HR and /APR Q&A area

Trips

Start a new trip / Create a trip
Prompts manual trip creation.


Start a trip with AI  / Create trip with AI  / Build a trip with AI
Prompts to use the trip builder across menus and banners.


Try AI trip builder / Try it
A more invitational way to encourage trip builder usage, especially when leaning into the recommendation engine over pure trip creation.
