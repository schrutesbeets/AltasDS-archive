Error messages

Guidelines and commonly used copy snippets featured across web and native.

An alert message that informs the user about technical errors or missing information required to proceed in a specific flow. Review this page for functional error messages used across the site, separated by use case. 

Length: Variable, but always prioritize clarity and simplicity of language over brevity.

Tone: Clear, plainspoken, but friendly when space allows. An upbeat, friendly tone is especially useful for technical errors or wait times (like on our AI-powered products).

CTAs: When possible, use buttons or in-line CTAs to trigger a reload or refresh. 

Looking for existing messages? Check the localization tool for existing errors as a start. You must be logged in to the VPN to access.

Common error messages

Some common messages used across the site include:

  • Oops! Something went wrong with your request. Try reloading this page.
  • Oops! There was a problem retrieving this content. Please try again.
  • Payment failed | Try again or use a different card or method.
  • Must be 18+ to check in | The traveler checking in must be 18 or older at the time of booking.
  • Scheduled maintenance | Service is temporarily paused. Please try again soon.
Look for opportunities to reuse assets in new spaces, especially since many of these already have translation keys. That said, no need to shoehorn an existing message into a space where a new language would work best.
