Best practices

 

Usage

Do

Use two or more Chips within a Filter Chip Row. Each Chip should represent a single category within the entire filter set.

Don't

Do not use a single Chip on its own. Use an Input Field, Dropdown Field or other input. Do not use Chips as a substitute for buttons. Avoid using Chips and Secondary Buttons in the same context, as they are nearly identical.

Don't

Do

A single Chip category may display a quantity of known results.

Don't

Do not modify the representation of result quantity in any way.

 

Variants

Do

Use the Filter Count variant of Chips to convey the quantity of 2 or more specific and related filters. Example: Number of travelers and rooms needed for filtering hotel options.

Don't

Do not modify the representation of filter quantity within Filter Count variants in any way.

Do

Use the Bubble Rating variants of Chips to convey a specific Bubble Rating to be applied when Filtering.

Don't

Do not modify the representation of Bubble Rating within Filter Count variants in any way. Do not modify chip contents to contain any elements besides icons, text labels, and Bubble Ratings.

Do

Use the default variant of Chips for all other permutations of icons and label text. Example: Filter Chip, Filter Chip with Filter Count, Dropdown Chip, Chip with Tooltip, Text Chip, Chip with Leading Icon, Chip with Leading Icon and Result Count

 

Active State

Do

Use the active state if the Chip contains content that can be applied when filtering. This applies to both default content as well as user inputs.

Don't

Do not use the active state for Chips whose contents cannot be applied when filtering.

 

Size

Do

Use the Chips in their supported height. Ensure all chips within a Filter Chip Row have the same height.

Don't

Do
