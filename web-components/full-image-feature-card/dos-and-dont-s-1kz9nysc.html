Full Image Feature Card ðŸ¥¶

A card featuring an image as the entire background.

  • Overview
  • Dos & Dont's

Best practices

 

Usage

Do

Use when wanting to strongly highlight a piece of content.

Don't

Donâ€™t use for standard content, or when content is limited.

 

Scaling

Do

On tablet web and desktop, use the Large variant expanded across all grid columns.

Do

Do not use the Large variants on mobile web. Do not limit the number of grid columns the Large variant spans on desktop.

Do

Full Image Feature Card is supported in a 3:4 Aspect Ratio until it grows to 500px in height. Once 501px in height, the container and image only expand in width.

Don't

Do not use the Full Image Feature Card in another aspect ratio when under 500px in height. Do no retain the 3:4 Aspect Ratio once the container grows to 501px or more.

 

Character Count

Do

Maintain the following maximums for character counts: 50 character title, 100 character subtitle and 25 character Button label. All text should wrap until there is no longer room within the card.

Don't

Do not exceed the max character counts.

 

Contents

Do

The Image, Button and title are required for all Full Image Feature Cards. All other elements are optional.

Don't

Do not configure a Full Image Feature Card without an Image, Button and/or title.
