Single Input Field ðŸ¥¶

Single Input Fields allow users to enter the number of guests, specify the number of rooms, or select a single date or a range of dates.

Also known as Input Entry Point, Input Box, Input

  • Overview
  • Dos & Don'ts

Best practices

 

Usage

Do

Use Atlas-sanctioned Picker Menus to capture the traveler's input.

Don't

Don't use custom menus for capturing traveler's input.

 

Size

Do

Single Input Fields remain the same size regardless of the length of the data entered.

Don't

Don't change the size of Single Input Fields.

 

Content

Do

The Single Input Field should update its value based on the selection from the associated Picker.

Don't

Don't allow users to freely add content. Unlike a Text Field, users cannot add content freely to an Single Input Field.

 

Default values

Specific features or scenarios can override this default (the override will be specified in the feature spec or design).

Do

Use "Today" as the default for the Date Field.

Don't

Do not allow ambiguous values in the Date Field.

Do

Use "8:00PM" as the default for the Time Field when referencing restaurant reservations.

Don't

Do not allow ambiguous values in the Time Field.

Do

Use "2" adults as the default for the Guest Field when referencing experiences or restaurant reservations.

Don't

Do not allow ambiguous values in the Guest Field.

Do

Use "1" room, "2" guests for the Stay Field when referencing a hotel reservation.

Don't

Do not allow ambiguous values in the Stay Field.
