Tooltip

Tooltips are small overlays that appear when users hover over, focus on, or tap an element. They provide context or explain the function of a piece of UI.

Also known as Screentip, Infotip, Hint

  • Overview
  • Dos & Don'ts

 

 

 

Types

There are three types of Tooltip categorized by what triggers it: Click, Hover and Static.

 

Title title Trigger click
Title title Trigger hover
Title title small Trigger static

 

Color Variants

There are three colors of Tooltip categorized by how much emphasis is needed: White, Green and GAI Black.

 

White Variant

The white Tooltip is usually triggered by the Tooltip icon.

Title title Trigger click
Title title Trigger hover
Title title small Trigger static

 

Green Variant

The green Tooltip usually appears when a user lands on the page. This Tooltip should be informational, provide guidance, or alert a user about a new feature.

Color green Title title Trigger click
Color green Title title Trigger hover
Color green Title title small Trigger static

 

Black Variant

The black Tooltip is reserved for the Generative AI product. It should only be used for GAI related pages, content and features.

Color black Title title Trigger click
Color black Title title Trigger hover
Color black Title title small Trigger static

 

 

Anatomy & Size

 

Anatomy - Tooltip Trigger

The Tooltip Trigger is always the Information icon that is top-aligned with its leading component or content.

 

Anatomy - Click ToolTip

The Click Tooltip contains a container, caret, title text, body text, circular button, and standalone link.

 

Anatomy - Hover ToolTip

The Hover Tooltip contains a container, caret, title text, and body text.

 

Anatomy - Static ToolTip

The Static Tooltip contains a container, caret, title text, and circular button.

 

Size - Tooltip Trigger

The Tooltip Trigger icon is 16x16px with Gap 05 (aka 4px) padding between its leading component or content and the trigger element.

Ensure there is a 44px44px tap target encasing the Tooltip Trigger in order to remain complaint with WCAG accessibility standards.

 

Size - Min/Max Widths

Caret positioning

The Tooltip has a variably positioned caret at the top, bottom, left, and right of the container. This caret can be positioned 24px from the left & right edge of the Tooltip container or anywhere between. The caretâ€™s position can be customized by adjusting the horizontal padding of the autolayout settings for the <caret position> layer.

 

 

 

Behavior

Most Tooltip open when a user hovers on an element. If a Tooltip includes a clickable action like a link or button, the Tooltip must be activated by a click instead of a hover for accessibility reasons. The third option is static. This tool tip includes alerts or informational dialogue messaging. Will be activated upon arrival. User always has the option to close.

On desktop, a hover Tooltip stays open for as long as the user hovers over the hover range around the point of origin. For a click Tooltip, the user can close the Tooltip by clicking the action items inside the Tooltip (if available) or by clicking anywhere outside the Tooltip. For a static Tooltip, the user can also close the Tooltip by clicking the action items inside the Tooltip.
On mobile, a user must click the point of origin to open either variants of the Tooltip. The user can close the Tooltip by clicking the action items inside the Tooltip (if available) or by clicking anywhere outside the Tooltip.

 

Hover Tooltip

A Tooltip activated on hover contains static text only (clickable actions inside a Tooltip cannot be accessed accessibly if the Tooltip opens on hover). On mobile, the hover Tooltip is activated by clicking the point of origin.

 

Click Tooltip

Tooltips activated through a click require a close button and can contain an optional link.

 

Static Tooltip

Tooltips activated upon arrival require a close button.

Accessibility

After the Tooltip opens, initial focus should be set on the first focusable element in the Tooltip. See WAI-ARIA Authoring Practices Modal Dialog Example, Accessibility Features section for best practices on setting focus.

Focus should not move if a Tooltip has been activated by hover. This will ensure we donâ€™t interrupt tab navigation.

Focus should move into the Tooltip if the Tooltip has been activated by a click (includes touch + keyboard), then restore focus to the icon when the Tooltip is closed.

The Tooltip componentâ€™s focus must not move outside the modal until it is closed.
